    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-azure-template-add-node" class="anchor" href="#azure-template-add-node" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>azure-template-add-node</h1>

<p>This ARM template create a set of worker nodes ready to be added to
existing CDH cluster using CM.</p>

<p>The template assumes the Virtual Network, Availability Set and Network
Security Group have been created before hand. </p>

<p><strong>Resource Naming Pattern</strong></p>

<ul>
<li>DNS prefix + copyIndex() is use to create a unique seed for naming
resources to avoid name collisions. </li>
<li>VirtualMachine name will be prefix-copyIndex()-suffix</li>
<li>StorageAccount will be uniqueString(seed+copyIndex())+"sa"</li>
<li>NetworkInterface will be uniqueString(seed+copyIndex())+"nic"</li>
</ul>

<p><strong>Caveat</strong></p>

<ol>
<li>dns prefix must be unique.</li>
<li>dns prefix can be slight different from each other per deployment, 
such as uniquestring1, uniquestring2...</li>
</ol>
